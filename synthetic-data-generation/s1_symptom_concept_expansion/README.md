# S1: Symptom Concept Expansion

Expand symptom concepts into fine-grained sub-concepts using an LLM. Takes symptom labels and brief DSM-5-based descriptions as input; produces a richer set of real-world expressions per symptom for downstream modeling.

## Input

Place `description_dictionary.json` in `load_dir` (default: `./input/`). Format: `{"SymptomName": ["desc1", "desc2", ...]}`

The bundled one is a temporary dictionary generated by Claude. The original used in the paper comes from **PsySym** (EMNLP 2022: *Symptom Identification for Interpretable Detection of Multiple Mental Disorders*); the authors provide the dictionary upon request.

## Run

```bash
# Step 1: Expand
python batch_api_for_generating_description.py --target depression

# Step 2: Format
python batch_api_for_generating_description_formatting.py
```

## Output

Files in `output/`: `batch_inputs_*.jsonl`, `batch_results_*.jsonl`, and `*.json`. 
The `.json` file is the expanded symptom description dictionary and is used in the next stage.
